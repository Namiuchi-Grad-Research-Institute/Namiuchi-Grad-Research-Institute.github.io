All rights reserved. * * Author: Sora Tonami * Created: 2021/08/12 * Last Modified: 2021/08/12 */ I. 本書の凡例について 本書で使用される凡例は次のように置き換えるべし。 AppName → アプリケーション名 appname → 同上 change → for文に於ける変化式 condition → 条件(結果が真偽値になるもの) dosomething → 何かしらの文 initialize → for文に於ける初期化式 pattern → case文に於ける定数式 variable → 何かしらの変数 II. ファイル形式について 1. 文字コードについて 文字コードは原則”UTF-8 (BOM無し)”とする。 2. 改行コードについて 改行コードは原則”LF (\n)(0x0A)(U+000A)”とする。 III. ソースコードについて 1. 命名規則について ｲ. 変数/メンバについて 変数、メンバは原則camelCase記法で命名するものとする。 但し、.NET系言語(Visual C# .NET等)はこの限りではない。 ﾛ. 定数/マクロについて 定数、マクロは原則UNDER_SCORE記法で命名するものとする。 ﾊ. 関数/サブルーチン/メソッドについて 関数、サブルーチン、メンバは原則camelCase記法で命名するものとする。 但し、.NET系言語(Visual C# .NET等)はこの限りではない。 ﾆ. クラス/インターフェース/列挙型について クラス、インターフェース、列挙型は原則Pascal記法(CamelCase記法)で命名するものとする。 ﾎ. 名前空間について 名前空間は原則Pascal記法(CamelCase記法)で命名するものとする。 ﾍ. パッケージ(Java)について パッケージ(Java)は原則全て小文字で命名するものとする。 2. 使用名前空間/パッケージ(Java)について 名前空間(.NET系言語)は”AccessMirai\NGRI\AppName”を原則として使うものとする。 名前空間(PHP)は”\AccessMirai\NGRI\AppName”を原則として使うものとする。 パッケージ(Java)は”com.accessmirai.ngri.appname”を原則として使うものとする。 3. if-else if-else文について | if(condition) { | dosomething(); | dosomething(); | } else if(condition) { | dosomething(); | dosomething(); | } else { | dosomething(); | dosomething(); | } のようにifやelseと始め括弧は間を開けず、閉じ括弧と始め中括弧は間を半角スペース1つとるものとする。 又、 | if(condition) dosomething(); | else if(condition) dosomething(); | else dosomething(); のように文が1つのみである場合1行にまとめるものとする。 4. while文について | while(condition) { | dosomething(); | dosomething(); | } のようにwhileと始め括弧は間を開けず、閉じ括弧と始め中括弧は間を半角スペース1つとるものとする。 又、 | while(condition) dosomething(); のように文が1つのみである場合1行にまとめるものとする。 5. for文について | for(initialize; condition; change) { | dosomething(); | dosomething(); | } のようにforと始め括弧は間を開けず、閉じ括弧と始め中括弧は間を半角スペース1つとるものとする。 括弧内のセミコロンは後方のみ半角スペース1つ入れることとする。 又、 | for(initialize; condition; change) dosomething(); のように文が1つのみである場合1行にまとめるものとする。 6. switch-case文について | switch(variable) { | case pattern: | dosomething(); | break; | case pattern: | dosomething(); | break; | default: | break; | } のようにswitchと始め括弧は間を開けず、閉じ括弧と始め中括弧は間を半角スペース1つとるものとする。 また、内容が空であっても原則default節は書くものとする。
